# 求解指引功能实施方案

> 版本：v1.0 | 状态：待执行
> 关联需求：求解指引功能需求文档 v1.1

---

## 整体策略

将实施拆分为 **两期**，优先落地可独立验证的基础设施，求解算法作为独立模块后续填入：

| 期次 | 内容 | 目标 |
|---|---|---|
| **第一期（当前）** | 编号系统 + 图片渲染 + 指令框架 | 所有前置基础就绪，可随时插入求解逻辑 |
| **第二期（后续）** | 求解算法（L1~L5） | 填入 `hint.ts`，`showHint` 直接生效 |

## 第一期实施计划

### Step 1 · 新建 `src/hint.ts`（编号 + 缓存，无求解）

**功能职责：**
- 前缀分配器
- `QuestionRecord` 缓存管理
- `showHint` 调用的对外接口（求解部分暂返回占位提示）

**数据结构：**

```typescript
// 一道题目的缓存数据
export type QuestionRecord = {
  puzzle: number[][];
  solution: number[][];
  targetRow: number;
  targetCol: number;
  targetAnswer: number;
  createdAt: number; // 毫秒时间戳
};

// 前缀注册条目
type PrefixEntry = {
  prefix: string;
  createdAt: number;
};
```

**对外暴露的方法：**

```typescript
export class HintManager {
  // 分配本轮前缀，每次新游戏开始时调用
  allocatePrefix(channelId: string): string

  // 注册题目到缓存，每道题生成后调用
  registerQuestion(channelId: string, questionId: string, record: QuestionRecord): void

  // 查询题目缓存（过期返回 null，未找到返回 null）
  getQuestion(channelId: string, questionId: string): QuestionRecord | null

  // 解析编号，分离前缀和题序（如 "ab3" → { prefix: "ab", index: 3 }）
  parseQuestionId(questionId: string): { prefix: string; index: number } | null

  // 求解接口（第一期返回占位提示，第二期替换为真实求解）
  solveHint(record: QuestionRecord): HintResult
}

export type HintResult =
  | { success: true; steps: HintStep[]; maxLevel: number }
  | { success: false; reason: "unsolvable" | "not_implemented" }

export type HintStep = {
  technique: string;
  level: number;
  description: string;
  eliminated: number[];
  remaining: number[];
}
```

**前缀分配算法：**

```
allocatePrefix(channelId):
  1. 取出该频道的 PrefixEntry[]（无则初始化为空数组）
  2. 找出所有 createdAt 超过 24h 的条目 → 可回收列表
  3. 若可回收列表非空：
       按字典序排序，取最小前缀
       更新该条目的 createdAt 为 Date.now()
       返回该前缀
  4. 若无可回收条目：
       取现有列表中字典序最大的前缀，递增得到新前缀
       若列表为空，从 "a" 开始
       追加新条目到列表
       返回新前缀
```

**26 进制前缀递增规则：**

```
"a" → "b"，"z" → "aa"，"az" → "ba"，"zz" → "aaa"
算法：末位字母 +1，若超出 z 则进位（等价于 26 进制加法）
```

### Step 2 · 修改 `src/renderer.ts`（底部布局）

**当前状态：** 底部只渲染 `level N` 居中。

**变更目标：** 底部同一行，左/中/右三段：

```
       level 2                    a3
（中间居中）                  （右对齐）
```

**函数签名变更：**

```typescript
// 新增可选参数 questionId
async renderBoard(
  puzzle: number[][],
  solution: number[][],
  targetRow: number,
  targetCol: number,
  difficulty: number,
  questionId?: string   // 新增，可选，不传则不渲染
): Promise<Buffer>
```

**底部渲染逻辑：**
- `level N` 文字绘制于底部行水平居中位置
- `questionId`（如 `a3`）绘制于底部行右侧，右对齐，与 `level N` 同字体同字号
- 若 `questionId` 未传入，底部行只渲染 `level N` 居中（向下兼容，不影响现有调用）

### Step 3 · 修改 `src/game.ts`（集成编号系统）

**3.1 `GameState` 类型新增字段：**

```typescript
currentPrefix: string;      // 本轮分配的前缀，如 "a"
currentQuestionIdx: number; // 当前是本轮第几题（1-based），用于拼题目编号
```

**3.2 `start()` 中分配前缀：**

```typescript
const prefix = this.hintManager.allocatePrefix(channelId);
newGame.currentPrefix = prefix;
newGame.currentQuestionIdx = 0;
```

**3.3 `askNextQuestion()` 中注册题目并传入编号：**

```typescript
game.currentQuestionIdx += 1;
const questionId = `${game.currentPrefix}${game.currentQuestionIdx}`;

// 注册到缓存
this.hintManager.registerQuestion(channelId, questionId, {
  puzzle: deepClone(puzzle),
  solution: deepClone(solution),
  targetRow: r,
  targetCol: c,
  targetAnswer: solution[r][c],
  createdAt: Date.now(),
});

// 渲染时传入 questionId
const image = await this.renderer.renderBoard(
  puzzle, solution, r, c, difficulty, questionId
);
```

**3.4 新增 `showHint()` 方法：**

```typescript
async showHint(session: Session, rawId: string): Promise<void>
```

处理逻辑：

```
1. rawId 非法（parseQuestionId 返回 null）
   → 回复「题目编号格式不正确，请输入如 a1、ab3 格式的编号」

2. 频道当前有游戏，且 rawId 的前缀 === game.currentPrefix
   → 回复「本轮游戏尚未结束，无法查询当前题目答案」

3. getQuestion(channelId, rawId) 返回 null
   3a. 该前缀从未出现过（parsePrefix 不在已知列表）
       → 「题目编号无效，请确认编号是否正确」
   3b. 该前缀存在但题目记录为空（可能已过期）
       → 「该题目已过期，无法查询」

4. 有记录 → 调用 solveHint(record)
   4a. success: false, reason: "not_implemented"
       → 「求解功能正在开发中，敬请期待」（第一期占位）
   4b. success: false, reason: "unsolvable"
       → 「此题超出当前支持的逻辑技巧范围，无法生成推理路径」
   4c. success: true
       → 格式化输出推理路径文本
```

## 第一期完成后的验证方式

| 验证项 | 方法 |
|---|---|
| 题目编号是否渲染在图片右下角 | 发起游戏，查看图片底部 |
| 难度标注居中是否正常 | 同上 |
| 每轮游戏是否用新前缀 | 连开两局，第一局用 `a`，第二局用 `b` |
| 24h 内同一前缀不重复 | 同一频道连续开局，编号应连续递增 |
| 游戏进行中查询当前轮 | 回复「本轮未结束」 |
| 游戏结束后查询 | 回复「求解功能开发中」（占位） |
| 查询不存在的编号 | 回复「编号无效」 |
| 指令名配置后生效 | 修改配置，重启，用新指令名测试 |

## 第二期预告（求解算法）

第一期完成后，第二期只需在 `hint.ts` 的 `solveHint()` 方法中填入具体逻辑，其他所有模块无需改动：

| 层级 | 技巧 | 实施顺序 |
|---|---|---|
| L1 | 宫排除、隐性唯余 | 第一批 |
| L2 | 行排除、列排除、区块排除 | 第一批 |
| L3 | 显性唯余、Naked/Hidden Pair/Triple | 第二批 |
| L4 | 区块唯余组合、数组唯余组合 | 第三批 |
| L5 | 多重复杂组合 | 第三批 |
