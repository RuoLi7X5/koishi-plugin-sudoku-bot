## 二、核心概念

### 2.1 两种出数类型

#### 显性唯余（Naked Single）
- **判定**：目标格本身的候选数集合 `size === 1`
- **推导视角**：看格子本身——"这格只能填X"
- **排除方向**：把8个非答案数字**从目标格**一一排除

#### 隐性唯余（Hidden Single）
- **判定**：在目标格所在的某行/列/宫中，答案数字只有目标格这1个位置能填
- **推导视角**：看整行/整列/整宫——"这一组里，只有这格能填X"
- **排除方向**：把答案数字**从同行/列/宫的其他8格**一一排除

> **关键区别**：显性唯余的已知数是目标格的"直接peer"（覆盖其他8个数字）；隐性唯余的已知数分散放置，用来将答案从同组其他格上消除，不要求是目标格的直接peer。两者的已知数数量和位置都不固定，取决于题目结构。

### 2.2 区块（Locked Candidate / 指向排除）

```
示意：目标格在第7行第9列，答案是5。
某个区块（如8宫 = 行7-9，列4-6）内，数字5的候选格
全部集中在第7行（如 (行7,列4) 和 (行7,列5)）。

这形成"锁定候选"：
  8宫内的5，只能在第7行 → 第7行在8宫范围外不受影响
  但等价地：第7行在其他宫内，5已被8宫的指向关系约束

实际排除效果（指向法）：
  8宫的5锁在第7行 → 第7行上，8宫以外有其他格想填5 → 这些格的5被排除
  （因为如果第7行上8宫外有格填5，则8宫的5就无法在第7行了）
```

区块用于：
- **显性唯余**：排除目标格的某个非答案候选数
- **隐性唯余**：排除同行/列/宫其他格上的答案候选

### 2.3 数对占位（Naked Pair Blocking，D5/D6）

```
示意：
  数对 {3,4} 在列5的第1行和第7行（即这两格候选数只有 {3,4}）
  → (行7,列5) 只能填3或4，不能填目标数1

  看第7行哪格能填数字1：
  · 宫7（列1-3）：宫内其他格已有1 → (行7,列1-3) 均排除
  · 宫8（列4-6）：(行7,列5) 被数对{3,4}占位不能填1；
                  (行7,列4)(行7,列6) 被其他已知数排除 → 宫8的行7全排除
  · 宫9（列7-9）：(行7,列7)(行7,列8) 被其他数排除
  → 只剩 (行7,列9) 可以填1 → 隐性唯余·行排除
```

数对占位的核心价值：数对**占据了一个位置**，使得某个格子不能填答案数，
进而间接形成或加强了一个区块效果，为最终的唯余出数创造条件。

## 三、难度3-6 设计规格

### 难度3 · 1区块 + 区块必要性

**指令**：`唯余训练 3`

**核心技巧**：1个区块（Locked Candidate），且该区块是推导出目标格的**必要条件**（去掉该区块，目标格就无法推出）

**出数类型**：随机为显性唯余或隐性唯余

| 题目形式 | 出数机制 | 已知数作用 |
|---------|---------|-----------|
| **显性唯余** | 目标格候选数减为1 | 1个区块排除目标格的1个非答案候选；其余直接peer格覆盖剩余非答案数字 |
| **隐性唯余** | 某行/列/宫中答案只能在目标格 | 1个区块排除同组某格的答案候选；其余已知数分散排除同组其他格的答案候选 |

**干扰数字**：有（目标数副本放在非peer区域 + 随机数），每步验证不产生新的出数

**示例（隐性唯余·行排除）**：
```
目标格：(行5, 列9)，答案：5
区块：宫6（行4-6，列7-9）内的5只在列9 → 列9在宫6外不能再填5
  → 行5的(行5,列9)以外：
     · (行5,列1-3) → 宫1/2已有5 → 排除
     · (行5,列4-6) → 宫5已有5 → 排除
     · (行5,列7-8) → 区块+其他数排除
  → 行5只有(行5,列9)能填5 → 隐性唯余
```

---

### 难度4 · 2区块 + 均必要

**指令**：`唯余训练 4`

**核心技巧**：2个独立区块，分别在不同行、不同列、不同宫，两个区块均为必要条件

**出数类型**：随机为显性唯余或隐性唯余

| 题目形式 | 出数机制 | 已知数作用 |
|---------|---------|-----------|
| **显性唯余** | 目标格候选数减为1 | 2个区块各排除1个非答案候选；其余已知数覆盖剩余非答案数字（直接peer格） |
| **隐性唯余** | 某行/列/宫中答案只能在目标格 | 2个区块各排除同组某格的答案候选；其余已知数继续排除同组其他格 |

**验证要求**：去掉区块1，无法推出目标格；去掉区块2，同样无法推出目标格。两个区块缺一不可。

**干扰数字**：有（同D3）

---

### 难度5 · 1数对→1区块 + 数对必要

**指令**：`唯余训练 5`

**核心技巧**：1个显性数对（Naked Pair）通过占位推导出1个区块，再由区块+已知数推出目标格

**技巧链**：
```
数对{P,Q}占据两格位置
  → 数对格不能填答案数A（占位效果）
  → 某宫内A被限制在特定行/列（形成区块）
    → 区块 + 其他已知数 → 目标格唯余（显性或隐性）
```

**出数类型**：随机为显性唯余或隐性唯余

**验证要求**：
- 去掉数对结构，区块无法形成 → 目标格无法推出（数对必要）
- 去掉区块，目标格无法推出（区块必要）

**数对来源**：逆向构造——先确定数对格的位置和数对数值{P,Q}，再通过在周围格填数使这两格候选数恰好被约束为仅{P,Q}

**干扰数字**：有（A的副本 + 随机数，逐步验证不破坏技巧链）

---

### 难度6 · 多数对/数组→2区块 + 均必要

**指令**：`唯余训练 6`

**核心技巧**：2个数对 或 1个三数组+1个数对，各自推导出1个区块（共2个区块），再由2区块+已知数推出目标格

**技巧链**：
```
数对/数组结构A → 区块1
数对/数组结构B → 区块2
  → 区块1 + 区块2 + 其他已知数 → 目标格唯余（显性或隐性）
```

**出数类型**：随机为显性唯余或隐性唯余

**验证要求**：数对/数组A、区块1、数对/数组B、区块2，任意去掉一环均无法推出目标格

**干扰数字**：有（同上，每步验证）

## 四、全局约束（所有难度均适用）

### 4.1 唯一出数约束（严格）

> **全盘只能推出目标格这1个数字，不允许任何其他格存在任何可推理的唯余（显性或隐性均不允许）。**

验证函数需要检查：
- 是否有其他显性唯余格（其他格候选数 === 1）
- 是否有其他隐性唯余格（其他行/列/宫中存在某数只有1格可填）
- 添加干扰数字后，是否新产生了上述情况

### 4.2 干扰数字添加规范

```
添加流程（逐步验证）：
1. 选择放置位置（非目标格，非必要的技巧结构格）
2. 放置数字
3. 立即验证：countDeducibleCells(puzzle) === 1 且唯一出数格 === 目标格
4. 验证失败 → 撤销此次放置，跳过
5. 验证通过 → 保留，继续尝试下一个干扰数字
```

### 4.3 技巧必要性验证规范

```
对每个技巧结构单元（区块/数对/数组）：
1. 临时移除该单元的关键格数字
2. 重新计算全盘可推理格
3. 若目标格仍可推出 → 该单元不必要 → 重新构造
4. 若目标格不可推出 → 该单元必要 ✓
```

## 六、生成算法详设

### 6.1 D3 显性唯余 构造流程

```
1. 随机选目标格 (TR, TC)，随机答案 A，随机"被区块排除的数字" D（D ≠ A）
2. 找一个宫 B：
   · 与目标格不同宫
   · 与目标格共行（BR = TR）或共列（BC = TC）
3. 在宫 B 内放置数字，使得 D 在宫 B 内的候选格全部在 TC 列（或 TR 行）
   · 形成区块：宫B的D仅在TC列 → TC列在宫B之外的格排除D（包括目标格）
4. 另选7个目标格的直接peer格（同行/列/宫），分别填入其余7个非A数字
   · 验证：目标格候选数 = {A}（显性唯余）
5. 验证全盘 isExactlyOneDeducibleCell(puzzle, TR, TC) === true
6. 添加干扰数字（逐步验证）
```

### 6.2 D3 隐性唯余 构造流程

```
1. 随机选目标格 (TR, TC)，随机答案 A
2. 随机选出数维度（行/列/宫），以"行TR"为例
3. 逐一阻断行TR其他8格填A的可能：
   · 7格：在各自的列或宫内放置A（A出现在该格的peer区域 → 排除该格的A）
   · 1格：通过区块（某宫内A锁定在TR行 → 该格作为区块来源格，TR行在其宫外确认排除）
4. 验证：行TR中只有(TR, TC)能填A（隐性唯余）
5. 验证全盘 isExactlyOneDeducibleCell(puzzle, TR, TC) === true
6. 添加干扰数字（逐步验证）
```

### 6.3 D4 构造流程（2区块）

```
基于D3流程扩展：
1. 构造区块1（同D3），排除目标格（或同组格）的第1个候选
2. 构造区块2（独立宫，不同行列宫），排除第2个候选
3. 验证两个区块均必要（分别移除检验）
4. 已知数覆盖剩余候选（显性）或排除同组其他格（隐性）
5. 全盘唯一性验证 + 干扰数字添加
```

### 6.4 D5 构造流程（数对→区块）

```
逆向构造：
1. 选目标格 (TR, TC) 和答案 A
2. 设计区块：宫C内数字A锁在TR行 → 对目标格行排除有效
3. 设计数对：找两格 (r1, c_pair) (r2, c_pair) 作为数对格，数对值 {P, Q}（P,Q ≠ A）
   · 数对格与区块形成关联：数对格占位后，宫C内只有TR行能填A（否则区块不成立）
4. 在数对格周围填数：使这两格候选数被约束为仅 {P, Q}（逆向推出需要哪些格填什么数）
5. 补充其余已知数（覆盖/排除剩余候选），形成完整题面
6. 验证链：移除数对 → 区块失效 → 目标格无法推出
7. 全盘唯一性验证 + 干扰数字添加
```

### 6.5 D6 构造流程（2数对/数组→2区块）

```
在D5基础上：
1. 构造数对/数组A → 区块1（同D5）
2. 独立构造数对/数组B → 区块2（不同宫，不同行/列）
3. 验证完整链：任意去掉一个数对/数组，对应区块失效，目标格无法推出
4. 全盘唯一性验证 + 干扰数字添加
```

## 七、实施计划（分阶段）

### 阶段1：基础架构重构（先行）
- `TrainingSession.mode` → `difficulty`（数字 1-6），保留旧字段兼容过渡
- `startTraining` 支持参数 1-6，路由到对应生成函数
- `fillTrainingPool` / `nextTrainingQuestion` 适配 difficulty
- 新增 `countHiddenSingleCells`、`isExactlyOneDeducibleCell`、`computeCandidates`
- 标签前缀、报告标题适配 difficulty 数字
- **本阶段结束后：D1/D2 功能完全保留，D3-D6 指令可识别但暂返回"建设中"提示**

### 阶段2：难度3实现
- `buildD3PuzzleOnce`（显性 + 隐性 两种路径，随机选择）
- 区块构造算法 + 区块必要性验证
- 干扰数字添加 + 逐步唯一性验证
- 集成测试：批量生成验证（目标：1000次，失败率 < 5%，生成耗时 < 500ms/题）

### 阶段3：难度4实现
- `buildD4PuzzleOnce`（双区块构造）
- 两区块独立性验证（确保各自必要）

### 阶段4：难度5/6实现（最复杂）
- 逆向构造框架（从技巧链到盘面）
- 数对构造算法（候选数约束推导）
- 完整链验证（数对→区块→目标格，逐环验证必要性）

| 项目 | 标准 |
|------|------|
