### 3. MVP系统

**评选规则**：
- 每局结束时自动评选MVP
- MVP = 本局得分最高且至少答对1题的玩家
- MVP在排行榜会显示👑图标

**MVP统计**：
- 记录每位玩家获得的MVP次数（mvpCount字段）
- MVP次数可触发成就：初露锋芒(1次)、常胜将军(10次)、传奇MVP(50次)

### 6. 数据库优化

**新增字段**：
- `perfectRounds`：完美局数（全对）
- `mvpCount`：MVP次数

**优化策略**：
- ❌ 不存储正确率（可通过 `totalCorrect / (totalCorrect + totalWrong)` 计算）
- ❌ 不存储当前排名（实时查询计算）
- ✅ 只存储无法计算的原始数据

## 🎯 使用场景示例

### 场景1：正常游戏流程
1. 玩家A输入：`数独开始`
2. 机器人发送数独图片
3. 机器人提问："第1题：A5格应该填什么？"
4. 玩家B输入：`7` ✅ 答对！获得10分
5. 机器人："恭喜 @玩家B 答对！+10 分（连续1次）"
6. ...8题结束后...
7. 机器人显示排行榜+MVP
8. 玩家解锁成就："🎊 恭喜 @玩家B 解锁成就【首战告捷】！获得 20 积分奖励！"

### 场景2：群嘲触发
1. 当前游戏有5位玩家参与
2. 第3题倒计时结束，无人答对
3. 机器人："集体摆烂了属于是，答案给你们：6"

### 场景3：单人嘲讽触发
1. 玩家C输入：`4` ❌ 答错！
2. 50%概率触发：
   - 触发："@玩家C 这都能错？离谱！-5分"
   - 不触发："@玩家C 答错了，扣 5 分。"

### 场景4：查询游戏进度
1. 游戏进行中，玩家输入：`游戏进度`
2. 机器人显示：当前题数、剩余时间、实时排名

## 🏆 荣誉头衔播报系统

### 播报格式

#### 授予头衔播报
当每局游戏结束后，系统会检测各项排行榜第一名并授予对应荣誉头衔：

```
👑 当前 [玩家名] [排行类型]排名第一！
🎖️ 小仙授予 [玩家名] 荣誉头衔「头衔名」
⏰ 有效期：7天
```

**示例**：
```
👑 当前 张三 积分排名第一！
🎖️ 小仙授予 张三 荣誉头衔「积分之王」
⏰ 有效期：7天
```

#### 易主播报
当检测到榜首发生变化时，会先播报易主信息，然后再播报授予新头衔：

```
🔄 [排行类型]榜首易主了！
[新玩家] 取代 [旧玩家] 成为新的第一！

👑 当前 [新玩家] [排行类型]排名第一！
🎖️ 小仙授予 [新玩家] 荣誉头衔「头衔名」
⏰ 有效期：7天
```

**示例**：
```
🔄 积分榜首易主了！
李四 取代 张三 成为新的第一！

👑 当前 李四 积分排名第一！
🎖️ 小仙授予 李四 荣誉头衔「积分之王」
⏰ 有效期：7天
```

### 五大荣誉头衔

| 头衔 | 排行类型 | 评判标准 | 有效期 |
|------|----------|----------|--------|
| 「积分之王」 | 积分 | 总积分最高 | 可配置（默认7天） |
| 「答题之王」 | 答对数 | 总答对题数最多 | 可配置（默认7天） |
| 「参与之王」 | 参与局数 | 参与游戏局数最多 | 可配置（默认7天） |
| 「开局之王」 | 开局数 | 发起游戏次数最多 | 可配置（默认7天） |
| 「正确率之王」 | 正确率 | 正确率最高（需满足10局参与门槛） | 可配置（默认7天） |

### 技术特点

1. **智能易主检测**：每局结束时自动检测榜首是否变化，仅在易主时播报特殊消息
2. **昵称获取**：尝试获取群成员昵称，失败时使用userId兜底
3. **符号规范**：荣誉头衔统一使用「」（半角书名号）包裹
4. **定时结算**：支持配置化的有效期，默认7天，可通过`titleDuration`调整
5. **自动更新**：每局游戏结束后自动触发头衔更新和播报
