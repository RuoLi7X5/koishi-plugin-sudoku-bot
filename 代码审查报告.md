### 6. **难度参数类型不一致**

**位置**: `index.ts` - 第103行

```typescript

    : undefined;
  return game.start(session, numDifficulty);
});
```

### 7. **题目选择随机性不够强**

**位置**: `game.ts` - `selectQuestions()` 方法（第713行）

```typescript
const shuffled = emptyCells.sort(() => Math.random() - 0.5);
```

**问题**: `Array.sort()` 配合 `Math.random() - 0.5` 的洗牌算法不够随机，存在偏差

**修复建议**: 使用 Fisher-Yates 洗牌算法
```typescript
private selectQuestions(puzzle: number[][], count: number): { row: number; col: number }[] {
  const emptyCells: { row: number; col: number }[] = [];
  for (let r = 0; r < 9; r++) {
    for (let c = 0; c < 9; c++) {
      if (puzzle[r][c] === 0) emptyCells.push({ row: r, col: c });
    }
  }
  
  // Fisher-Yates 洗牌
  for (let i = emptyCells.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [emptyCells[i], emptyCells[j]] = [emptyCells[j], emptyCells[i]];
  }
  
  return emptyCells.slice(0, count);
}
```

### 8. **排行榜未处理同分情况**

**位置**: `game.ts` - `showRank()` 方法（第296-298行）

**体验问题**: 同分玩家可能每次查询排名都不同

**修复建议**: 增加次要排序条件
```typescript
sorted = (users as any[])
  .sort((a, b) => {
    const diff = b[selected.field] - a[selected.field];
    if (diff !== 0) return diff;
    // 同分按答对数排序
    return b.totalCorrect - a.totalCorrect;
  })
  .slice(0, 10);
```

### 10. **中间件性能问题**

**位置**: `index.ts` - 第151-156行

```typescript
ctx.middleware(async (session, next) => {
  if (session.content && /^[1-9]$/.test(session.content)) {
    await game.handleAnswer(session, parseInt(session.content));
  }
  return next();
});
```

**问题**: 
- 中间件会拦截**所有**消息，即使没有游戏在进行
- `game.handleAnswer()` 内部会检查是否有游戏，但每条消息都要走一遍

**性能影响**: 在高频聊天的群里，可能影响其他插件响应速度

**修复建议**: 在中间件中先快速检查是否有游戏在进行
```typescript
ctx.middleware(async (session, next) => {
  if (session.content && /^[1-9]$/.test(session.content) && game.hasGameInChannel(session.channelId)) {
    await game.handleAnswer(session, parseInt(session.content));
  }
  return next();
});
```

需要在 `SudokuGame` 类中添加：
```typescript
hasGameInChannel(channelId?: string): boolean {
  return this.currentGame?.channelId === channelId;
}
```

### 13. **MVP判定规则不透明**

**位置**: `game.ts` - `endGame()` 方法（第532-538行）

```typescript
let mvpUserId: string | null = null;
for (const [uid, data] of sorted) {
  if (data.correct > 0) {
    mvpUserId = uid;
    break;
  }

```

**问题**: 
- MVP只要求"得分最高且答对至少1题"
- 如果最高分是负分（一直答错），则无MVP
- 但用户不知道这个规则

**建议**: 
- 在文档中明确说明MVP判定规则
- 或在无MVP时说明原因（如"本局无人答对，无MVP"）

### 14. **答错后立即公布答案**

**位置**: `game.ts` - `handleAnswer()` 方法（第442-457行）

**问题**: 
- 玩家A答错后，只扣分不公布答案
- 其他玩家仍可继续抢答
- 但答错的玩家不知道正确答案是什么

**体验问题**: 挫败感强，学习反馈弱

**建议**: 
- 方案1: 答错后在群里公布答案（但会剥夺其他玩家的答题机会）
- 方案2: 答错后私聊告知正确答案（保护其他玩家体验）
- 方案3: 答题时间结束后统一公布，答错的玩家稍后才知道

当前实现是方案3，相对合理，但可以考虑增加私聊反馈
